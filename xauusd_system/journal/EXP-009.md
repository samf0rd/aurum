# EXP-009: M15 combined levers — confirmation=3 + donchian=40

**Date:** 2026-06-07
**Status:** COMPLETE — combination does not stack; confirmation=3 remains dominant
**Results file:** results/exp-009.json
**Experiment config:** experiments/exp-009.yaml

## Hypothesis

If EXP-007 (conf=3) adds +0.12 PF and EXP-008 (d=40) adds +0.05 PF independently,
combining both might reach PF ~0.95 overall.

## Result

| Metric | EXP-007 (conf=3) | EXP-008 (d=40) | EXP-009 (both) |
|--------|-----------------|---------------|----------------|
| Trades | 1,708 | 1,722 | 1,413 |
| PF | 0.90 | 0.83 | **0.88** |
| Net P&L | -$49,373 | -$73,618 | -$53,479 |
| Max DD | -71.4% | -76.9% | -68.9% |

**EXP-009 is worse than EXP-007 alone.** The combination doesn't stack — adding
donchian=40 on top of confirmation=3 slightly hurts performance (0.90 → 0.88)
while reducing trade count by 17%.

TRENDING_BULL PF exactly 1.00 in EXP-009 (vs 1.06 in EXP-007). The wider channel
neutralizes confirmation=3's bull-regime gain. The interaction likely filters some
high-quality TRENDING_BULL breakouts that would trigger on d=20 but not d=40.

Strip-top-3 PF 0.81 — worse than EXP-007 (0.83) and far below EXP-010 (0.97).
False-breakout rate 77.0% — identical to EXP-007 (not improved by wider channel).

## Conclusion

The two levers are non-additive. Donchian=40 is an inferior configuration for
every metric vs donchian=20 when confirmation=3 is already applied. **Donchian=20
with confirmation=3 (EXP-007 / EXP-010) is strictly better than donchian=40.**

The optimal parameter set so far is EXP-010: `long_only=True, conf=3, d=20, ADX≥25`.
